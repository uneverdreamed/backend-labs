Тестирование CORS-политик

Клиент: http://localhost:5001 → API: http://localhost:5000 (разные порты = разные origin = кросс-доменные запросы)

1. Students — политика "AllowClient" (разрешён localhost:5001)

Все методы должны работать, потому что наш origin (5001) разрешён.

2. PublicCourses — политика "ReadOnly" (только GET с любого origin)

GET должен работать. POST будет заблокирован — политика разрешает только GET.

3. Courses — GET работает, DELETE отключён через [DisableCors]

GET пройдёт (политика по умолчанию). DELETE заблокирован атрибутом [DisableCors].
